# 图检测规则与源码复核标准

分析器输出的是候选项，不是确定问题。置信度表示图证据强度，不表示重构必要性。

## `orphan-symbol` 疑似无调用方

### 图规则

符号节点没有行为型入边，并且未被标记为入口或公开导出。

### 必查项

- 路由与框架注册
- 动态导入、反射和依赖注入
- JSX、模板、配置文件和字符串引用
- 包导出和对外 API
- 测试、脚本和构建工具引用
- 约定式文件名入口

只有完成上述检查后才能给出“建议移除”。否则使用“证据不足”。

## `single-call-wrapper` 单一转发封装

### 图规则

节点只有一个行为型出边，同时至少存在一个行为型入边。

### 保留理由

- 参数归一化或类型转换
- 权限、审计、日志和埋点
- 缓存、重试、限流和熔断
- 错误转换与协议适配
- 稳定的领域命名或模块边界
- 便于测试替换的接口

函数体确实只做无语义转发时，才能建议合并或移除。

## `similar-dependency-neighborhood` 重复职责

### 图规则

两个符号至少共享两个行为依赖，邻居集合 Jaccard 相似度不低于 0.72。

### 必查项

- 两者是否解决同一业务问题
- 是否因不同业务规则而独立变化
- 参数、返回值和错误语义是否一致
- 是否具有不同副作用
- 调用顺序和条件分支是否一致
- 抽象后是否需要大量布尔参数或模式分支

如果抽象会引入模式分支或让两个独立变化方向绑定，结论应为“建议保留现状”。

## `high-coupling-hub` 高耦合节点

### 图规则

行为型入度与出度之和达到当前图的高位阈值，最小阈值为 6。

### 必查项

- 节点是稳定基础设施还是业务聚合体
- 是否包含多个独立变化原因
- 是否跨越数据、展示、权限、网络等职责
- 高入度是否仅说明它是公共能力
- 拆分是否会增加更多不稳定接口

公共类型、事件总线和稳定工具函数不能只因连接数高就判定为问题。

## `community-boundary-leak` 社区边界泄漏

### 图规则

节点至少有三个跨社区行为型出边，且占全部行为型出边的 60% 以上。

### 必查项

- Graphify 社区是否与真实模块对应
- 该节点是否属于适配层、聚合层或编排层
- 是否绕过公开接口访问内部实现
- 是否存在反向依赖或不稳定依赖方向

社区命名错误或聚类失真时，不生成确认问题。

## `import-cycle` 循环依赖

### 图规则

文件导入图存在包含两个以上文件的强连通分量。

### 必查项

- 导入是否为类型导入
- 构建工具是否会消除该依赖
- 是否产生模块初始化顺序问题
- 循环是否跨越架构层级
- 移动共享类型能否解除循环

只有确认存在运行时或架构风险时才提高严重度；类型级循环通常不应直接判为阻塞。

## 置信度使用约束

| 图置信度 | 含义 | 允许结论 |
|---|---|---|
| `>= 0.85` | 结构关系明确 | 仍需源码复核，可形成任意结论 |
| `0.60-0.84` | 有较强结构信号 | 需要补充源码和调用证据 |
| `< 0.60` | 弱候选 | 默认保留或证据不足，除非源码证据充分 |
