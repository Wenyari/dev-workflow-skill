# artifact 层（规划中）

承载跨环节的数据产物，方法论定义两个：

- **未闭环清单**（open-issues）：三个评审环节里所有"没当场闭环的疑点"的统一收口
- **校准清单**（calibration）：自测阶段发现的遗漏 → 归因 → 反哺给 PRD↔UI 语义检查流程

## 分层原因

未闭环清单、校准清单跨多个 skill 读写：

- `prd-ui-check` 输出未认领项 → 未闭环清单
- 自测阶段发现遗漏 → 校准清单
- 校准清单回喂 → 修正 `prd-ui-check` 的语义判断规范

多方读写必须独立成层，不能挂在任何单一 skill 下。

- `calibration/` 未启动
