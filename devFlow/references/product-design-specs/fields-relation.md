# 关联选择字段规范

## 跨表选取

### 适用范围

- 项目、合同、供应商、收款账户、付款账户等。

### 录入方式

- 下拉单选、弹窗单选、下拉多选、弹窗多选、快捷操作直接写入。
- 默认值：无，或按规则自动计算。
- 输入限制：按需设置是否支持输入。

### 字段说明

- 仅选取：提示文本为“请选择”。
- 支持输入：提示文本为“请选择/输入”。
- 自动计算：提示文本为“选择XX后自动带出”。
- 帮助文本：一般无帮助文本，如有要求按需说明。

### 列表显示

- 按需显示。
- 筛选：模糊查询。
- 高级筛选：等于、不等于、包含、不包含、为空、不为空。
- 排序：按字母升序或降序。

### 产品要点

- 必须明确单选或多选。
- 选择关键信息时，需要明确辅助信息，尤其在关键信息可能重复时，例如重名用户增加工号、部门、岗位。
- 必须明确选项是否全量显示。
- 必须明确选项数据范围，以及是否与用户数据权限关联。
- 必须明确来源数据值变更后，本表数据如何处理：实时更新或快照冻结。

### UI 与体验要点

- 辅助信息较少或选项较少时，优先使用下拉选取。
- 辅助信息较多或来源数据量较大时，优先使用弹窗选取。
- 一般单选场景回显选项值即可。
- 多选且信息简单时，以 Tag 形式回显。
- 多选且信息复杂时，以表格形式呈现。
- 选项较多时，支持关键词检索。
- 不支持输入时，输入信息仅用于关键词模糊查询，失焦后清空输入框。
- 多选时，已选项支持删除，全部选项支持清空。

### 研发要点

- 选项列表返回结构需要包含关键字段及辅助信息。
- 存储选项 ID，而不是选项值。
- 大数据量场景需要支持分页加载或虚拟滚动。

## 用户选择

### 适用范围

- 项目经理、销售人员等。

### 录入方式

- 用户选择器，支持单选或多选。
- 选项范围按需设置条件，例如仅在职用户、仅业务销售部用户。
- 默认值：无、当前用户、按规则自动计算。

### 字段说明

- 提示文本：请选择。
- 帮助文本：一般无帮助文本。

### 字段校验

- 无。

### 列表显示

- 按需显示，一般显示用户头像和用户名。
- 筛选：下拉多选。
- 高级筛选：等于、不等于、包含任意一个、不包含、同时包含、为空、不为空。
- 排序：按用户名拼音升序或降序。

### 产品要点

- 必须明确人员范围：全公司、本部门、指定部门、指定角色。
- 必须明确是否支持选择已离职人员。
- 必须明确人员变更后历史数据处理方式：保留原值或同步更新。

### UI 与体验要点

- 显示用户姓名和头像。
- 悬浮显示部门、岗位等辅助信息。
- 支持关键词检索。
- 多选时采用穿梭框，可见已选项。
- 多选回显使用 Tag，支持逐个删除和一键清空。
- 已离职人员需有明确标识，例如灰色或标签。

### 研发要点

- 存储用户 ID，显示时关联查询用户姓名等信息。
- 需要支持根据角色、部门、岗位过滤可选人员列表。

## 部门选择

### 适用范围

- 所属部门、归属组织、成本中心等。

### 录入方式

- 部门选择器，支持单选或多选。
- 选项范围按需设置。
- 默认值：无、当前用户所属部门、按规则自动计算。

### 字段说明

- 提示文本：请选择。
- 帮助文本：一般无帮助文本。

### 字段校验

- 无。

### 列表显示

- 按需显示。
- 筛选：下拉多选。
- 高级筛选：等于、不等于、包含任意一个、不包含、同时包含、为空、不为空。
- 排序：按部门编码或部门名称拼音升序/降序。

### 产品要点

- 必须明确选择范围。
- 必须明确部门变更后历史数据处理方式：保留原值或同步更新。

### UI 与体验要点

- 支持关键词检索。
- 多选时采用穿梭框，可见已选项。
- 多选回显使用 Tag，支持逐个删除和一键清空。
- 已撤销部门需有明确标识，例如灰色或标签。

### 研发要点

- 存储部门 ID，显示时关联查询部门名称。
